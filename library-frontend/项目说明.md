# 图书管理系统 - 完整项目说明

## 项目概述

这是一个完整的前后端分离的图书管理系统，包含：

- **后端**: Spring Boot 3.2.0 + MySQL
- **前端**: Vue 3 + Element Plus

## 项目结构

```
项目根目录/
├── library-backend/          # 后端项目
│   ├── src/
│   │   └── main/
│   │       ├── java/         # Java 源代码
│   │       └── resources/    # 配置文件
│   ├── pom.xml
│   └── README.md
│
└── library-frontend/         # 前端项目
    ├── src/
    │   ├── api/              # API 接口
    │   ├── views/            # 页面组件
    │   ├── router/           # 路由配置
    │   ├── store/            # 状态管理
    │   └── utils/            # 工具函数
    ├── package.json
    └── README.md
```

## 快速开始

### 1. 启动后端

```bash
cd library-backend
# 确保 MySQL 已启动并创建了 library 数据库
mvn spring-boot:run
```

后端服务运行在: `http://localhost:8080`

### 2. 启动前端

```bash
cd library-frontend
npm install
npm run dev
```

前端服务运行在: `http://localhost:3000`

### 3. 访问系统

1. 打开浏览器访问 `http://localhost:3000`
2. 首次使用需要注册账号
3. 注册后登录系统
4. 开始使用各项功能

## 开发注意事项

1. **数据库配置**: 修改 `library-backend/src/main/resources/application.yml` 中的数据库连接信息
2. **API 代理**: 前端已配置代理，开发时自动转发到后端
3. **CORS**: 后端已配置允许跨域请求
4. **密码加密**: 后端使用 BCrypt 加密密码

## 常见问题

### 1. 后端启动失败
- 检查 MySQL 是否启动
- 检查数据库 `library` 是否已创建
- 检查数据库连接配置是否正确

### 2. 前端无法连接后端
- 确保后端服务已启动
- 检查后端端口是否为 8080
- 检查浏览器控制台错误信息

### 3. 登录后无法访问页面
- 检查路由守卫配置
- 检查用户角色权限
- 清除浏览器缓存和 localStorage

## 部署说明

### 后端部署
1. 打包: `mvn clean package`
2. 运行: `java -jar target/library-backend-0.0.1-SNAPSHOT.jar`
3. 配置生产环境数据库

### 前端部署
1. 构建: `npm run build`
2. 将 `dist` 目录部署到 Web 服务器（如 Nginx）
3. 配置 API 代理或修改 API 地址
